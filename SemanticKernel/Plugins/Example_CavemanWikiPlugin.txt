🔧 How to Register the Plugin with Semantic Kernel

using Microsoft.SemanticKernel;
using caveman.core;
using caveman.core.SemanticKernel.Plugin;

// 1. Build your kernel
var builder = Kernel.CreateBuilder();

// 2. Add your AI services (example: Azure OpenAI)
builder.AddAzureOpenAIChatCompletion(
    deploymentName: "gpt-4",
    endpoint: "https://your-resource.openai.azure.com/",
    apiKey: "your-api-key");

// 3. Option A: Register with default internal instances
builder.Plugins.AddFromType<CavemanWikiPlugin>("CavemanWiki");

// 4. Option B: Register with custom/pre-configured instances (recommended for dependency injection)
var compressionService = new CavemanCompressionService();
var wiki = new CavemanWiki(); // Optionally configure wiki settings here
builder.Plugins.AddFromObject(new CavemanWikiPlugin(wiki, compressionService), "CavemanWiki");

// 5. Build the kernel
Kernel kernel = builder.Build();

🤖 How AI Can Invoke the Plugin
Example 1: Full Wiki Generation
csharp

var result = await kernel.InvokeAsync<string>(
    pluginName: "CavemanWiki",
    functionName: "generate_project_wiki",
    new KernelArguments
    {
        ["projectPath"] = "C:\\Users\\Dev\\MyApi",
        ["compressionLevel"] = 2,
        ["maxFileSizeKB"] = 100,
        ["includeContents"] = true
    });

// result now contains the markdown string
Console.WriteLine(result);

Example 3: Validate Path First

var detection = await kernel.InvokeAsync<string>(
    pluginName: "CavemanWiki",
    functionName: "detect_project_type", 
    new KernelArguments { ["projectPath"] = userInputPath });

// Parse the JSON-like response
if (detection.Contains("\"isValid\": true"))
{
    // Safe to proceed with full wiki generation
    var wiki = await kernel.InvokeAsync<string>(/* ... */);
}
else
{
    // Inform user about invalid path
    return "That doesn't appear to be a valid project folder.";
}

Chain Multiple Calls for Complex Tasks

User: "How do I add authentication to my Python project?"

AI Workflow:
1. [detect_project_type] → Confirm it's Python ✅
2. [get_project_summary] → Get dependencies (check for Flask/Django) ✅  
3. [generate_project_wiki] with includeContents=true → Get file structure ✅
4. Synthesize answer using: project type + existing deps + file locations


Handle Errors Gracefully

var result = await kernel.InvokeAsync<string>(/* ... */);

if (result.StartsWith("[ERROR]"))
{
    // Extract clean error message for user
    var errorMsg = result.Replace("[ERROR] ", "");
    return $"I couldn't access that project: {errorMsg}";
}

// Proceed with normal processing


⚙️ Configuration Options
Via Constructor (Advanced)

// Customize CavemanWiki behavior before plugin registration
var wiki = new CavemanWiki();
// Note: CavemanWiki uses static properties for configuration
// Example: CavemanWiki.IncludeExtensions = new[] { ".cs", ".py", ".md" };

var plugin = new CavemanWikiPlugin(wiki, compressionService);
builder.Plugins.AddFromObject(plugin, "CavemanWiki");


Via Function Parameters (Runtime)
All main parameters are exposed to the AI:
projectPath: Which folder to scan
maxFileSizeKB: Control token budget (10-1000 KB)
compressionLevel: Balance detail vs. conciseness (0-3)
includeContents: Toggle heavy file content inclusion
